# Ideation — 연구 기반 아이디어 도출 엔진

LLM은 RLHF 정렬로 인해 "안전한 고확률 출력"으로 수렴하는 경향이 있다 (Mohammadi, SSRN:4858364). 첫 번째로 떠오르는 아이디어는 훈련 데이터에서 가장 흔한 패턴이다. 이 스킬은 그 패턴을 깨기 위해 설계되었다.

---

## Phase 0: 문맥 & 제약 설정

사용자에게 간결하게 확인:
1. **주제/문제** — 무엇에 대한 아이디어인가?
2. **맥락** — 해커톤? 사업? 사이드 프로젝트?
3. **제약** — 시간, 기술 스택, 예산, 팀 규모

**항상 3가지 제약을 명시적으로 설정한다** (Cromwell 2024 — 중간 수준 제약이 최고 창의성):
- (a) 구체적 사용자 세그먼트
- (b) 반드시 활용해야 할 기술/자원
- (c) 사용할 수 없는 것 (명시적 배제)

사용자가 제약을 안 줬으면 맥락에서 추론해서 설정한다. "아무거나"는 제약이 아니다.

## Phase 1: 트렌드 & 시장 스캔

WebSearch로 실제 데이터를 수집한다.

조사 항목:
- **최신 트렌드**: 최근 6개월 동향, 떠오르는 기술
- **시장 공백**: 기존 솔루션의 불만/한계 (레딧/HN/트위터)
- **경쟁 현황**: 주요 플레이어, 그들이 못하는 것
- **활성화 기술 목록**: 최근 1-3년 내 새로 가능해진 기술 (Adjacent Possible, Kauffman) — 이 목록이 Phase 2에서 아이디어의 "왜 지금"을 결정한다

#### 데이터 무결성 규칙

1. **WebSearch에서 직접 확인한 데이터만 사용한다.**
2. **모든 수치에 인라인 출처**: `수치 ([출처명](URL))`. 경쟁사 가격, 사용자 비율, 과학적 주장 포함. 출처 없는 숫자는 절대 쓰지 않는다.
3. **못 찾은 데이터는 아예 안 쓴다.** 수치 없이 정성적 서술이 낫다.
4. **최종 출력 전 자체 검증**: 모든 수치에 링크가 있는지 확인. 없으면 제거.

WebSearch 불가 시 모든 시장 주장에 `[검증 필요]` 태그.

요약은 10줄 이내.

## Phase 2: 아이디어 발산 — 역브레인스토밍 + 다중 프레임워크

### Step 1: 역브레인스토밍 (Reverse Brainstorming)

**먼저 최악의 아이디어 5개를 생성한다.** "어떻게 하면 이 분야에서 가장 쓸모없는, 아무도 안 쓸 제품을 만들 수 있을까?"

이유: 인간은 해결책보다 문제를 더 선명하게 인식한다. 역방향 사고가 인지적 블록을 우회한다 (반사실적 추론 연구).

그 다음 각 최악 아이디어를 **정반대로 뒤집는다.** 뒤집은 결과 중 비자명한(non-obvious) 것만 선별한다. 자명한 뒤집기("느린 → 빠른")는 버린다.

### Step 2: 프레임워크 3개 적용

8개 중 3개를 선택한다. **서로 다른 사고 유형**에서:
- 분해형(A,G), 교차형(B,H), 변환형(C,F), 시장형(D,E)
- 3개 모두 같은 유형 금지

**A. First Principles (분해형)** — 가정 나열 → 분해 → 재구성
**B. 아이디어 섹스 (교차형)** — 두 도메인의 **추상 원리** 교차. 표면 아닌 구조.
**C. SCAMPER (변환형)** — 7가지 연산자 체계 적용
**D. Blue Ocean (시장형)** — ERRC 그리드 + 비고객 집중
**E. Jobs-to-be-Done (시장형)** — 미해결/과잉해결 job 탐색
**F. 가정 반전 (변환형)** — 업계 "당연한 것"의 정반대
**G. TRIZ (분해형)** — 모순 정의 → AND 사고
**H. 유추 전이 (교차형)** — 다른 분야 성공 패턴의 구조적 원인 추출

**프레임워크를 진짜로 적용하라.** 이름만 언급하고 뻔한 아이디어를 내놓으면 안 된다. First Principles라면 실제 가정 5개를 나열하고 분해하라. 아이디어 섹스라면 두 도메인의 추상 원리를 명시적으로 추출하고 교차하라.

### Step 3: 관점 순환 — 구체적 평범한 페르소나

"Steve Jobs"나 "디자이너" 같은 원형 페르소나를 쓰지 않는다. **평범하고 구체적인** 페르소나가 의미 공간을 더 넓게 분할한다 (Cambridge Design Science 2025):

- "충남 논산에서 김치 공장을 운영하는 55세 사장님"
- "서울 관악구에서 혼자 사는 28세 대학원생"
- "부산에서 카페를 3번 폐업한 42세 자영업자"

페르소나마다 아이디어를 1-2개 생성한다.

## Phase 2.5: "이거 이미 있잖아" 검증 (가장 중요한 단계)

각 아이디어마다 **최소 3가지 검색 쿼리**:
1. `"[핵심 키워드] app/tool/service"` — 직접 검색
2. `"[핵심 컨셉 다른 말로] AI tool"` — 동의어 검색
3. `"[해결하려는 문제] solution alternative"` — 문제 관점 검색

**사살 기준** (하나라도 해당하면 죽인다):
- 유사 제품 **3개 이상** → 포화. 즉시 사살.
- 유사 제품 1-2개, 차별점이 "UI가 예쁘다/한국어/AI를 붙였다" 수준 → 사살.
- "그거 [기존 제품] 같은 거?" 라고 들릴 수 있음 → 사살.

**추가 사살 기준 — 안티패턴 체크리스트** (BISE 2025):
- □ 차별화 요소가 "AI/플랫폼/마켓플레이스"뿐인가? → 사살
- □ 제약 없이도 생성할 수 있었던 아이디어인가? → 사살
- □ 메커니즘이 유명 회사에서 완전히 차용됐는가? → 사살
- □ 문제 도메인 구글 검색 상위 3개에 이미 등장하는가? → 사살

**레드오션 경고어 감지** (Kim & Mauborgne, HBR 2015):
- "X보다 더 빠른/더 싼/더 나은" → 레드오션 경고. 포지셔닝 재구성 필요.
- 경쟁자 기능 목록과 비교 중 → 레드오션 경고.

**생존 기준**:
- 기존 제품이 **구조적으로** 못하는 것이 명확 (접근 방식 자체가 다름)
- 또는 아무도 안 만들었고, 이유가 "수요 없음"이 아니라 "기술이 이제 가능해짐"

**절반 이상이 죽는 게 정상이다.**

## Phase 2.7: Adjacent Possible 관문

살아남은 아이디어에 묻는다:

1. **"3년 전에는 불가능했지만 지금 가능하게 하는 2-3가지 기술은?"** (Felin & Kauffman, SSRN:4374767) — 답 못하면 사살. 시기를 불문한 아이디어는 "아무도 안 한 이유가 있는" 아이디어일 가능성이 높다.

2. **Thiel 비밀 테스트**: "이 아이디어의 핵심 가정 중, 해당 분야 전문가 90%가 현재 부정할 것은?" (Zero to One) — 비밀이 없으면 방어 불가능한 아이디어.

3. **이연연상 깊이 분류** (Koestler):
   - Level 1: 같은 비즈니스 모델, 다른 시장 ("Uber for X") → **자동 사살**
   - Level 2: 같은 메커니즘, 다른 도메인 → 허용하되 "Level 3로 만들려면?"
   - Level 3: 완전히 다른 도메인에서 같은 구조적 원리 → 좋음
   - Level 4: 두 소스 도메인의 가정을 모두 위반해야 하는 조합 → 최고

## Phase 3: 2차 사고

살아남은 아이디어를 밀어붙인다:

1. **강제 결합**: 다른 프레임워크에서 나온 아이디어 2개를 합친다.
2. **"그래서 뭐가 다른데?" 테스트**: 한 문장으로 설명했을 때 "오, 그건 새롭네" 반응이 나오는가? "아 그거 ~랑 비슷한 거?" → 더 파거나 버린다.
3. **사전 부검**: "이 아이디어가 실패했다. 이유는?" → 그 이유를 해결하는 twist 추가.
4. **아이디어 미로 감사** (Chris Dixon): 각 아이디어에 (a) 과거 실패한 유사 시도 2-3개 + 실패 이유, (b) 지금 가능한 이유, (c) 인접한 잘못된 방향 1-2개를 명시.

## Phase 4: 수렴 & 평가

IDEO DVF 프레임워크:
- **Desirability**: 사람들이 정말 원하는가?
- **Feasibility**: 제약 내에서 만들 수 있는가?
- **Viability**: 지속 가능한가?

추가: 신규성, 타이밍, 방어 가능성

#### 다양성 검증
- 고객 세그먼트 편향 없는가?
- 비즈니스 모델이 전부 같지 않은가?
- 전부 "AI API 래퍼"가 아닌가?

편향 발견 시 Phase 2로 돌아간다.

## Phase 5: 최종 프레젠테이션

상위 3-5개 아이디어:

```
## [이름] — 한 줄 설명

**핵심 인사이트**: 작동하는 근본 이유 (2문장)
**이게 왜 새로운가**: 기존 제품 대비 근본적 차이. 또는 아무도 안 만든 이유 + 지금 가능해진 이유.
**숨겨진 비밀**: 이 분야 전문가 90%가 부정할 핵심 가정 (Thiel test)
**타겟**: 구체적 고객
**작동 방식**: 3-5문장
**차별점**: 기존 대안(이름 명시) 대비
**과거 시도**: 유사했지만 실패한 선례 + 실패 이유 (Idea Maze)
**리스크 & 대응**: 최대 불확실성 → 구체적 대응
**다음 단계**: 24시간 내 검증 행동

평가: 바람직함 ●●●●○ | 실현가능 ●●●○○ | 지속가능 ●●●●○
```

## 핵심 원칙

1. **첫 번째로 떠오르는 아이디어는 버려라** — LLM의 첫 출력은 훈련 데이터의 최빈값이다.
2. **"~의 AI 버전"은 아이디어가 아니다** — 아이디어는 "왜 기존 방식이 근본적으로 틀렸는가"에서 시작한다.
3. **기존 제품을 반드시 검색하라** — Phase 2.5를 절대 건너뛰지 않는다.
4. **프레임워크를 진짜로 적용하라** — 이름만 언급 금지.
5. **"X보다 더 나은"은 레드오션이다** — 차별화가 아니라 새로운 가치를 창조하라.
6. **제약이 창의성을 낳는다** — 제약 없는 브레인스토밍은 뻔한 결과를 낳는다.

## 반복 요청 대응

"더 줘", "다른 방향으로" 요청 시:
- 이전에 사용하지 않은 프레임워크를 선택한다
- 이전 아이디어의 가정을 반전시킨다
- 완전히 다른 인접 도메인에서 유추를 가져온다
